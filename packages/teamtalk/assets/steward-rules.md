# Global Rules

These are the team's always-on rules. Each entry is a summary; the full
content lives at the linked MemFS path under `team/rules/global/`.

## think-before-coding

**Path:** `team/rules/global/think-before-coding.md`
**Tags:** [thinking, process]
**Summary:** State assumptions and surface tradeoffs before writing
code. Do not begin implementation when requirements are uncertain;
ask rather than guess.

## verify-before-done

**Path:** `team/rules/global/verify-before-done.md`
**Tags:** [quality, completion]
**Summary:** Define success criteria up front and verify against them
before declaring work complete. Run lint, type-check, and tests where
the project provides them.

## surface-tradeoffs

**Path:** `team/rules/global/surface-tradeoffs.md`
**Tags:** [communication, decision-making]
**Summary:** When presenting alternatives, explain the reasoning behind
the recommendation. Do not silently choose between valid options.
